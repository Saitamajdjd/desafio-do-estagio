# Envio de Mensagens WhatsApp via Supabase + Z-API

Script em Python que lê contatos cadastrados no Supabase e envia, via Z-API,
a mensagem personalizada `"Olá, <nome> tudo bem com você?"` para até 3 contatos.

## Funcionalidades

- Busca até 3 contatos no Supabase, ordenados pela coluna `time`.
- Personaliza a mensagem com o nome de cada contato.
- Envia via Z-API (WhatsApp).
- Modo `DRY_RUN` para testar sem enviar mensagem real.
- Logs estruturados de cada etapa.

## 1. Setup da tabela no Supabase

Crie uma tabela chamada `contatos` com as colunas:

| Coluna     | Tipo                                  | Descrição                                  |
|------------|----------------------------------------|---------------------------------------------|
| `id`       | `bigint` (PK, identity)                | Identificador único do contato               |
| `nome`     | `varchar(100)`                         | Nome do contato (usado na personalização)    |
| `telefone` | `varchar(20)`                          | Telefone com ou sem DDI (ex: `55719...`)     |
| `time`     | `timestamp with time zone` (`now()`)   | Data de cadastro, usada para ordenar a busca |

SQL de criação:

```sql
create table public.contatos (
  id bigint generated by default as identity primary key,
  nome varchar(100) not null,
  telefone varchar(20) not null,
  time timestamp with time zone default now()
);
```

### Row Level Security (RLS)

A tabela deve ter RLS habilitado, permitindo apenas **leitura (SELECT)** pela
chave `anon`. O projeto usa a chave `anon`/`public` no `.env` — nunca a
`service_role` — para que o RLS configurado seja efetivamente respeitado.

```sql
alter table contatos enable row level security;

create policy "permitir leitura anon"
on contatos
for select
to anon
using (true);
```

## 2. Variáveis de ambiente

Crie um arquivo `.env` na raiz do projeto com:

```
SUPABASE_URL=https://SEU-PROJETO.supabase.co
SUPABASE_KEY=SUA_CHAVE_ANON_PUBLIC

ZAPI_INSTANCE_ID=sua_instance_id
ZAPI_TOKEN=seu_token
ZAPI_CLIENT_TOKEN=seu_client_token

DRY_RUN=true
```

- `SUPABASE_URL` / `SUPABASE_KEY`: encontrados em **Project Settings → API**
  no painel do Supabase. Use a chave **anon / public**.
- `ZAPI_INSTANCE_ID`, `ZAPI_TOKEN`, `ZAPI_CLIENT_TOKEN`: obtidos no painel da
  [Z-API](https://www.z-api.io/), após criar uma instância gratuita.
- `DRY_RUN`: defina como `true` para apenas logar as mensagens (sem enviar de
  fato) ou `false`/remova para enviar via WhatsApp de verdade.

## 3. Instalação

```bash
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate

pip install -r requirements.txt
```

## 4. Como rodar

```bash
python main.py
```

- Com `DRY_RUN=true`: o script busca os contatos e mostra no log as
  mensagens que seriam enviadas, sem chamar a Z-API.
- Com `DRY_RUN=false`: o script envia as mensagens reais via WhatsApp para
  até 3 contatos cadastrados na tabela `contatos`.
